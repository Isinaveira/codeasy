# Proposal: Import Code from AI Assistant

## Intent

Allow users to import code blocks generated by the AI assistant directly into the editor panels with a controlled diff+confirm flow, instead of manual copy-paste.

## Scope

### In Scope
- Parse ` ```html`, ` ```css`, ` ```js`/` ```javascript` blocks from AI markdown responses.
- "Import" button overlay on each detected code block (styled like a copy button).
- Modal with diff view comparing current editor code vs suggested code.
- Two-click flow: 1st click opens modal → 2nd click "Accept changes" applies the code.
- Toast notification (3-4s) on successful import.
- `Ctrl+Z` undo notice in the modal.
- Web mode: map blocks to corresponding HTML/CSS/JS editors.
- Algorithms mode: JS blocks target the algorithms editor, same diff flow.

### Out of Scope
- Importing to multiple editors with a single click.
- Editing the imported code inside the modal before accepting.
- Persistent undo history beyond Ctrl+Z.

## Capabilities

### New Capabilities
- `code-import`: Parse, diff, and import code blocks from AI messages into editors.

### Modified Capabilities
- None

## Approach

1. **Code Block Parser**: Utility function `parseCodeBlocks(markdown: string)` that extracts `{ language, code }[]` from markdown, handling ` ```html`, ` ```css`, ` ```js`, ` ```javascript`.
2. **ChatInterface update**: Render each code block in the assistant's message with an "Import" button overlay. Clicking opens a confirmation modal.
3. **Diff Modal**: Show side-by-side or unified diff between current editor content and suggested code. Include a note about `Ctrl+Z`. "Accept changes" button triggers the replacement.
4. **Store actions**: Use `useCodeStore` setters (`setHtml`, `setCss`, `setJs`) or add them if missing. In algorithms mode, target the `js` field.
5. **Toast**: Show a brief success toast after accepting.

## Affected Areas

| Area | Impact | Description |
|------|--------|-------------|
| `src/components/AiAssistantPanel/ChatInterface.tsx` | Modified | Render code blocks with Import button |
| `src/components/AiAssistantPanel/CodeBlock.tsx` | New | Code block component with Import overlay |
| `src/components/AiAssistantPanel/ImportDiffModal.tsx` | New | Diff modal with accept/confirm |
| `src/utils/parseCodeBlocks.ts` | New | Markdown code block parser |
| `src/store/useCodeStore.ts` | Modified | Ensure html/css/js setters are available |

## Risks

| Risk | Likelihood | Mitigation |
|------|------------|------------|
| Markdown parsing misses edge cases | Low | Use regex that handles optional language metadata and closing fences |
| Diff is confusing for beginners | Medium | Keep it simple: show current (left) vs suggested (right) with clear labels |

## Rollback Plan

Revert changes to ChatInterface.tsx, useCodeStore.ts, and remove new components. No data migration needed.

## Success Criteria

- [ ] Code blocks with html/css/js languages show an Import button.
- [ ] First click opens a modal with a diff view.
- [ ] Second click (Accept) replaces the editor code.
- [ ] Toast appears after import.
- [ ] Ctrl+Z undoes the import.
